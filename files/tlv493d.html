esp_err_t update()

Function to read out values from tlv493d.

Note

This is a blocking function. It try to read out data. If the tlv493d is not ready or new new data will present, it will immediately retry to read out. If it will not get new data after ‘READOUT_MAX_TIMES’ it will return as ESP_FAIL.

Returns:

ESP_OK when success.

TLV493D power modes

Definition of power modes for tlv493d

enum tlv493d_mode_t

Enum for powermodes.

Note

MASTERCONTROLLERMODE and MASTERCONTROLLERMODE_ULP are not supported for the moment

Values:

enumerator POWERDOWNMODE

Used for disabling tlv493D. It will use 3nA in this mode(see datasheet).

enumerator FASTMODE

In this mode the value is only 8 bit accuracy but the highest read speed.

enumerator LOWPOWERMODE

Read out data ever 12ms

enumerator ULTRALOWPOWERMODE

Read out data ever 100ms

enumerator MASTERCONTROLLERMODE

Read out data controlled be cpu. This mode has the same speed as fastmode. This mode can be used interrupt driven or polled.a64l

Note

At the moment only polled mode can be enabled until we find a solution to connect sda pin to interrupt routine. Maybe this can be done by ‘gpio matrix’.

enumerator MASTERCONTROLLERMODE_ULP

Warning

Is not supported yet!

TLV493D read out register

Here we can found the read register

BANK_X1 0x00

[7:0] x significant bits 11…4

BANK_Y1 0x01

[7:0] y significant bits 11…4

BANK_Z1 0x02

[7:0] z significant bits 11…4

BANK_TEMP1 0x03

[7:4] temp significant bits 3…0

[3:2] frame count

[1:0] channel

BANK_XY2 0x04

[7:4] x siginificant bits 3…0

[3:0] y siginificant bits 3…0

BANK_Z2 0x05

[7] not used

[6] testmode flag

[5] parity check

[4] power down flag (must be 1 if complete otherwise reading is running)

[3:0] z significant bits 3..0

BANK_TEMP2 0x06

temperature significant bits 11…4

BANK_RES1 0x07

[6:3] must be written from register BANK_Z2

BANK_RES2 0x08

[7:0] must be written into register REG_MOD2

BANK_RES3 0x09

[4:0] must be written into register REG_MOD3

TLV493D write register

Here we can found the write register

REG_MOD0 0x00

reserved

REG_MOD1 0x01

[7] parity bit: Sum of all 32 bits from write registers 0 H , 1H , 2 H and 3H must be odd.

[6:5] i2c address bits: Bits can be set to “00”, “01”, “10” or “11” to define the slave address in bus configuration. Caution! must be filled from BANK_RES1

[4:3] must be filled from BANK_RES1

[2] interupted enabled

[1] fast mode

[0] low power mode

REG_MOD2 0x02

fill witch BANK_RES2

REG_MOD3 0x03

[7] Temperature measurement enabled

[6] Low-power period “0” period is 100ms (ultra low-power period), “1” period is 12ms

[5] Parity test enabled

[4:0] must be filled from BANK_RES3
